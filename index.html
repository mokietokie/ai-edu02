MCP — 도구를 꽂는 표준

Model Context Protocol · Anthropic 2024년 말 발표
❌ 기존

AI는 자기가 학습한 것만 안다. 외부 세계와 단절.

🟢 실습 레퍼런스 CHEAT SHEET A

VBA 자동화 4 STEP — 한눈에

STEP 1 업무를 쪼갠다
"내가 손으로 뭘 뭘 하고 있지?" 순서대로 적기
→ 결과: 모듈 A, B, C, D, E (5개 내외)
STEP 2 모듈 하나씩 GPT에게
"전체 자동화 만들어줘" 금지
→ 짧은 VBA 코드 5개
STEP 3 각 모듈 독립 테스트
반드시 테스트 파일로! (원본 금지)
에러 → 그대로 GPT에 붙여넣기
STEP 4 메인 프로시저로 통합
Call 모듈A → Call 모듈B → ...
→ 버튼 한 번으로 끝
🚨 원칙 : 한 번에 전체 ❌ / 한 모듈씩 ✅
이 슬라이드는 실습 블록 내내 "기준점" 역할. 청중이 어느 단계인지 헷갈릴 때마다 이 화면을 가리키며 "지금 STEP 2입니다" 하는 식.

🟢 실습 레퍼런스 CHEAT SHEET B

프롬프트 3원칙 & 에러 대처

프롬프트 3원칙
원칙 ①
역할을 부여하라
"너는 10년차 VBA 전문가야"
원칙 ②
맥락(Context)을 제공하라
"Sheet1 A열에 날짜, B열에 거래처..."
원칙 ③
출력 형식을 지정하라
"한국어 주석 달아줘"
에러 났을 때 대처 흐름
  • ① VBA 실행 → 빨간 창 등장
  • ② 오류 번호 + 메시지 복사
  • ③ GPT에 그대로 붙여넣기 + "수정 코드 알려줘"
  • ④ 수정본 받아 재실행
  • ⑤ 또 오류? → 반복 🔁

💡 신입사원에게 업무 지시하듯 구체적으로.
프롬프트 = AI에 대한 업무 지시서.

실습 중 청중이 "너무 추상적으로 물어봤다" 싶을 때 이 슬라이드로 돌아와서 원칙 어긴 지점을 짚어주면 효과적.

BLOCK 6 VBA'S LIMITS

VBA의 한계 — 더 나아가야하는 이유

✅ VBA가 잘하는 것
  • Excel · Word · PPT · Outlook 내부 작업
  • 로컬 파일 처리
  • 사내 네트워크 파일 조작
